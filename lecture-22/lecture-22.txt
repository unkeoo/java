What is dynamic Blocks

Java has automatic memory management

how does java DeAllocate blocks
1 Garbage Block
Garbageblocks:
=================
Are those dynamic blocks whyich are not being reffered by any reference 

int[] arr=new int[5];     // this array will be not connected to anything after the new array is made, turned into garbage block


arr=new int[10]; 


2 Garbage Collector 

a. Inside Jvm we have a special software running which is called Garbage


b. the major responsibility of the GC is to identify and recollect those dynamic blocks which have become garbage blocks.

c.All this activity keeps on executing in the backround. Thus as a developer we never come to know when JVM activated GC
, when it Collected the garbage blocks and how the memory was freed 

d.Thats the reason we say in Java memory management ( memory DeAllocation) is always automatic
without any interference of the programmer.

e. This gc is a very very lazy software.
